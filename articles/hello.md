# 一般公開の経緯について
時の流れは早いもので、この本を出版したのは2020年の5月30日と今から遡ること約４年前の出来事です。当時はコロナウイルスによる様々な影響があり、出版予定から約一ヶ月を過ぎてから出版がされました。そのため2024年の４月末をもって出版契約の有効期限の更新日がやってきました。

そこで同著者である　[やぬすさん]()の提案、[にたさん]()の同意により、『**契約を解除し一般公開をする**』という決断をすることになりました。この件に関しては出版元である**技術評論社様**にも相談をさせていただき、わがままを了承をいただきました。
この場を借りて、深くお礼申し上げます。

## なぜ公開することにしたのか？？
大きな理由としては２点あります。
一つは『**技術書であるため、４年もの歳月の経過により内容がかなり陳腐化してしまっため、現在にお勧めし難く、流通を取り止めたい**』ということと、
『**執筆の目的はReact Nativeの普及促進であること**』という二点です。
そのため、一般公開することとなりました。

## なぜ一部なのか・・・？？
技術の進歩は非常に目覚ましいもので、その一方非常に残酷なもので・・・・
執筆当時から、４年前で更新が止まっているコードベースが現代においてどのようになっているのか・・・というと
- react-navigation v5が出たばっかりだったのに現代では7系が準備されていたり
- 素のreduxが全盛期でtool-kitやrecoilなどの面々はまだ実践投入するには尖り過ぎている時代だったり
- typescriptが3系だったのが現代では5系になっていたり

などなど・・・・
コードが絡んできたりするような箇所は軒並みアップデートしなければならず『**そのまま公開は無理そう・・・😅』という結論になりました🙇‍♂️　
よって現段階においても通用する箇所のみの公開ということで一部公開となりました。

公開していない部分については、どうなるかは未定ではありますが、現在においてはお蔵入りをさせる予定でいます。

# 執筆の経緯について
本記事も含めてせっかくの機会ですので、経緯や執筆について書かせてください。
執筆の発端はReact Native Japanというコミュニティをしていたところ、当時の代表者に担当者の西原さんから執筆の依頼が来たことが発端になります。

その後、技術評論社本社へ足を運ばせていただき、こんな編集社ってこんな感じなんだ・・・！！と緊張しながらお話をお伺いさせていただいた記憶があります。

当時はまだまだReact Native関連の書籍がほとんど流通しておらず、React Native関連の書籍を出版したいとのことでした。

本のレベル感や対象者については、僕らの方で設定させていただくことになり、
『**現場でしっかりと使っているエンジニアが提供する**』
『**僕らの目的はReact Nativeの普及促進である**』
ということから、『**入門書ではなく現場でしっかりと使える実践的な内容**』という書籍にし、『**React Nativeを実戦投入してもらう機会を増やす**』という目的をかかげました。

こうして執筆が開始されるのですが・・・・一度ある程度書き終えるものの・・・内部で色々あったり・・・・体を壊したり・・・・・などなどの事情が重なり、執筆が頓挫してしまいました・・・・（西原さんこの件については本当に申し訳ございませんでした。）

その後、執筆を再始動させ、無事に今日に至ることができました🙌


# 商業誌執筆の経験について
## 良かった点
同人誌は出版したことがあるものの商業誌については初めてのことで非常に勉強になりました。

同人誌については、内容・ページ量・レイアウトなどなど、誰かに任せることはできるけども自分が執筆者が全ての裁量を持ってできる反面、自分で全てをなんとかしなければなりません。

これに対して、商業誌では、さすが出版社！ということもあって、図やレイアウトに関してはとても手厚いサポートを受けることができます。また、表現方法や構成などについてもしっかりと添削をしていただくことができました！（一部の図については、身内デザイナーに手伝っていただきました）

特に印象深かったのは、同人誌などでエンジニア同士で添削するとしたら、github作ってレビューする感じになる・・・つまりテキストベースでの添削が行われるかと思うんですが、PDFで帰ってくるという点でした。文章をお送りすると、出版物の同様のPDFが作成され、そこに指摘が直接書かれているという感じです。

## 大変だった点
反対にしんどかったこととしては、内容とちょうどいいボリューム感です。
商業誌の場合はターゲットをなるべく広くして、しっかりを内容を詰め込みたいものです。
しかし・・・誌面である以上、やはりページ数には制限があるため、何でも間でも詰め込むというわけにはいきません。
同人誌でも同じようなことが発生するのですが、同人誌の場合は巻を分けたり、特定のライブラリなどのテーマのみで発刊すれば良いのに対して、商業誌となると別巻を最初から出すのが難しかったり、各書ある程度まとまりのあるテーマや内容である必要があったりなどなど、『一つの商業誌』としての体を成すための制限というものがあります。

とりわけ僕の場合は、『これも書きたい・・・！！』『これも書かないと・・・！！』などなど、最初に詳しく書きすぎていたため**泣く泣く大幅にカット**しなければならないということがありました🥲　

そして、これに対して『**もっと詳しく書いてほしかった**』という趣旨のコメントをいただいたのですが『**むしろカットしたんだよぉぉぉ**😭』と少し悲しい気持ちになりました・・・。

少し気張った意見にはなるのですが、やはりこうした大衆に向けて発信をしていくに向けて何が良い形かを検討しなければならないのは、商業誌ならではの難しさだと非常に実感しました。

たとえば、本にはページ数の制限があるのでReact Navigationにのみ注力した『React Navigation詳解』という分厚い本を出したとして
- React Nativeをこれからやる人が興味を持つか？
- どれくらいの人が興味がある？

などなどなると、結論としては『ほとんどの人が興味がない』といったところになります(僕は即買いです・・・！！）。となると、React Nativeの地点である程度すでにマイナーなため、React Native全般にページを割り当てていくことならざるを得ません（執筆の目的もReact Native普及のためですし）。
この点が同人誌とは大きく異なるポイントで、商業誌にそのような尖りまくった本がないにはそういった所以で、僕がもどかしく感じたポイントでもあります。

## 商業誌執筆のメリット
金銭面の話でいくと、正直なところそこまで美味しいものではないですが、実際に印税をいただけるとなると色々勉強になり、『○リー○ッター』の印税って、とんでも無いことよりもとんでも無いことになってるんだなとしみじみ思いました・・・笑。さらには、本の値段と流通数の関係だったりとかもかなり勉強になりました。
なんか勉強になりましたばっか書いているんですが、本当にここら辺はかなり勉強になり見識が広がったところでした！！

そして、肝心な実利的なメリットなのですが、先ほども申し上げた通り、直接的な金銭メリットはないのですが、その反面、キャリアとしては非常に良い武器になりました。
この点は非常によく、直接お仕事をいただく機会も増え、転職の際も有利に働き・・・・などなど、日々忙しく過ごさせていただけるほどさまざまな機会をいただくことができました。

## 商業誌執筆のデメリット
皆さんもすでにお察しの通り、書いた後のデメリットはありません（ちゃんと申告はしましょう）！！

強いていうならば執筆時しんどいということです。
商業誌ともなると『まとまった一つのテーマ』にたいしてしっかりと書いていかないといけないので各分量にしては非常に大変です。しかもライブラリのアップデートやら、ブレイクスルーの登場やら、なんやらで、技術書はゆっくり書いていていいものではありません。
なので・・・・『**一人で書かれている方、マジですごい・・・！！**』とおもいました。
（色々揉めたのもあるんですが・・・笑）僕らの場合は一年以上要しました（もちろんアップデートには追従をしています）。
